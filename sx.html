The Language

sx compiles to native code through LLVM. It’s designed to stay close to the machine while keeping the ergonomics modern — expressive types, compile-time metaprogramming, and clean C interop.

#import "modules/std.sx";

Point :: struct {
    x, y: i32;
    magnitude :: (self: *Point) -> f32 {
        sqrt(self.x * self.x + self.y * self.y)
    }
}

main :: () {
    p := Point.{ x = 3, y = 4 };
    print("point: {}, magnitude: {}\n", p, p.magnitude());
}
#import "modules/std.sx";

// generics via monomorphization
max :: (a: $T, b: T) -> T {
    if a > b then a else b
}

// first-class closures, captured by value
make_adder :: (n: i64) -> Closure(i64) -> i64 {
    return closure((x: i64) -> i64 => x + n);
}

main :: () {
    add5 := make_adder(5);
    print("{}\n", max(3, 9));    // 9
    print("{}\n", add5(100));   // 105
}

Native via LLVM

Compiles straight to optimized native binaries through LLVM, with selectable optimization levels and target CPUs.

Compile-time execution

Run code at compile time with #run and generate code with #insert — metaprogramming without a separate macro language.

Modern type system

Generics by monomorphization, protocol-based polymorphism, optionals, tagged-union enums, and pattern matching.

First-class closures

Closures capture by value; bare functions auto-promote to closures when a closure type is expected.

C interop

Bind C functions with the extern linkage modifier, or import headers and sources directly with #import c.

Cross-compilation

One toolchain targets macOS, Linux, Windows and WebAssembly — ARM64 and x86_64.

Native on iOS & Android

Talk to the platform directly. sx has built-in intrinsics for Objective-C and JNI, so you call UIKit, Foundation and the Android SDK from sx — no hand-written bridging glue.

iOS — Objective-C

// define a real Objective-C class, in sx
Counter :: #objc_class("Counter") {
    count: i32;
    bump :: (self: *Self) { self.count += 1; }
}

// call any Cocoa / UIKit method by selector
#objc_call(void)(view, "setBackgroundColor:", color);

Define Obj-C classes in sx — ivars, properties and methods, with ARC (autorelease, strong and weak) — and call any Cocoa or UIKit method by selector with #objc_call.

Android — JNI

// call Java through JNI — method-ids cached, types mapped
#jni_env(env) {
    win := #jni_call(*void)(activity,
        "getWindow", "()Landroid/view/Window;");

    px := #jni_static_call(i32)(color,
        "rgb", "(III)I", 255, 0, 64);
}

Reach Java through #jni_call and #jni_static_call inside a #jni_env block — method-ids are cached and Java primitives map to sx types for you.

Get started

sx is open source under the MIT license. Build the compiler with Zig and LLVM.

# build the compiler
zig build

# compile and run
sx run file.sx

# compile to a native binary
sx build file.sx -o app

# cross-compile
sx build app.sx --target wasm
